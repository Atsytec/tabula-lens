# Security Policy

## Supported Versions

Security updates are provided for the latest released version of Tabula Lens.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately to us before disclosing it publicly.

**How to Report:**

- Email: info@tabula-lens.dev
- Include "Security Vulnerability" in the subject line
- Provide as much detail as possible: reproduction steps, affected versions, impact assessment

**What to Expect:**

- We will acknowledge receipt within 5 business days
- We will provide a preliminary response within 14 business days
- We will work with you to understand and validate the issue
- We will coordinate a fix and release timeline

**Disclosure Process:**

- We will privately fix the vulnerability
- We will release a security update as soon as possible
- We will announce the vulnerability publicly after the fix is released
- We will credit you in the security advisory (with your permission)

## Security Best Practices

Users of Tabula Lens should:

- Keep dependencies updated
- Review security advisories in our GitHub releases
- Follow secure coding practices when using the library
- Never expose database credentials to the frontend (this is a core design principle of Tabula Lens)
