## Key Principles

Tabula Lens is built around several key architectural principles:

- **HTTP API as Universal Interface** - All components communicate via a well-defined HTTP API
- **Modularity** - Clear separation between frontend, backend, and database layers
- **Interchangeability** - Frontend and backend implementations can be swapped independently
- **Framework Agnostic** - The HTTP API can be consumed by any frontend framework
- **PostgreSQL First** - PostgreSQL is the primary supported database

## Architecture Decision Records

Architecture Decision Records (ADRs) document significant architectural decisions, including:

- Why the HTTP API was chosen as the universal interface
- Why PostgreSQL was selected as the primary database
- Why the component architecture follows a modular pattern
- And other key design decisions
