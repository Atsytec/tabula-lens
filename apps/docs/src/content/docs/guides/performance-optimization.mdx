# Performance Optimization Guide

This guide covers performance optimization strategies for Tabula Lens applications, including database optimization, backend API performance, and frontend rendering optimization.

## Overview

Performance optimization in Tabula Lens applications involves three key areas:

1. **Database Performance** - Query optimization and indexing
2. **Backend API Performance** - Caching, connection pooling, and response optimization
3. **Frontend Performance** - Rendering optimization, data fetching, and bundle optimization

## Database Performance

### Query Optimization

#### 1. Use Appropriate LIMIT Values

```javascript
// ✅ Good: Reasonable page size
const result = await tabulaLens.query({
  table: 'users',
  page: 1,
  limit: 50  // Optimal for most use cases
});

// ❌ Avoid: Too large
const result = await tabulaLens.query({
  table: 'users',
  page: 1,
  limit: 10000  // Will cause performance issues
});
```

#### 2. Implement Database Indexing

```sql
-- Create indexes on frequently filtered columns
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_created_at ON users(created_at);
CREATE INDEX idx_users_status ON users(status);

-- Composite index for multiple column queries
CREATE INDEX idx_users_status_created ON users(status, created_at);
```

#### 3. Use Selective Column Queries

```javascript
// ✅ Good: Select only needed columns
const result = await tabulaLens.query({
  table: 'users',
  columns: ['id', 'name', 'email'],  // Only select needed columns
  limit: 50
});

// ❌ Avoid: SELECT *
const result = await tabulaLens.query({
  table: 'users',
  limit: 50  // Returns all columns
});
```

### Connection Pooling

#### Configure Connection Pool

```javascript
const { Pool } = require('pg');

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  max: 20,                    // Maximum pool size
  min: 5,                     // Minimum pool size
  idleTimeoutMillis: 30000,    // Close idle connections after 30s
  connectionTimeoutMillis: 2000,  // Timeout for new connections
});

const tabulaLens = new TabulaLens(pool);
```

#### Pool Size Guidelines

```javascript
// Calculate optimal pool size
// Formula: (connections * utilization) / query_time

// Example: 4 CPU cores, 0.8 utilization, 10ms query time
const optimalPoolSize = Math.floor((4 * 0.8) / 0.01);  // ~320 connections

// For most applications: 10-20 connections is sufficient
const pool = new Pool({
  max: 20,  // Good starting point
  min: 5    // Keep some connections ready
});
```

### Database Caching

#### Application-Level Caching

```javascript
const NodeCache = require('node-cache');
const cache = new NodeCache({ 
  stdTTL: 300,  // 5 minutes
  checkperiod: 60  // Check for expired keys every 60s
});

app.get('/api/tabula-lens', async (req, res) => {
  const cacheKey = `tabula:${JSON.stringify(req.query)}`;
  const cached = cache.get(cacheKey);
  
  if (cached) {
    return res.json(cached);
  }
  
  const result = await tabulaLens.query(req.query);
  cache.set(cacheKey, result);
  res.json(result);
});
```

#### Redis Caching

```javascript
const Redis = require('ioredis');
const redis = new Redis(process.env.REDIS_URL);

app.get('/api/tabula-lens', async (req, res) => {
  const cacheKey = `tabula:${JSON.stringify(req.query)}`;
  const cached = await redis.get(cacheKey);
  
  if (cached) {
    return res.json(JSON.parse(cached));
  }
  
  const result = await tabulaLens.query(req.query);
  await redis.setex(cacheKey, 300, JSON.stringify(result));  // 5 minutes
  res.json(result);
});
```

## Backend API Performance

### Response Compression

```javascript
const compression = require('compression');

app.use(compression({
  filter: (req, res) => {
    if (req.headers['x-no-compression']) {
      return false;
    }
    return compression.filter(req, res);
  },
  threshold: 1024  // Only compress responses > 1KB
}));
```

### Response Pagination

```javascript
app.get('/api/tabula-lens', async (req, res) => {
  const { page = 1, limit = 50 } = req.query;
  
  // Validate pagination parameters
  const validatedLimit = Math.min(Math.max(limit, 1), 100);  // Max 100 per page
  const validatedPage = Math.max(page, 1);
  
  const result = await tabulaLens.query({
    ...req.query,
    page: validatedPage,
    limit: validatedLimit
  });
  
  // Add pagination metadata
  res.json({
    data: result.data,
    pagination: {
      page: validatedPage,
      limit: validatedLimit,
      total: result.total,
      totalPages: Math.ceil(result.total / validatedLimit)
    }
  });
});
```

### Query Batching

```javascript
// Batch multiple queries into a single request
app.post('/api/tabula-lens/batch', async (req, res) => {
  const { queries } = req.body;
  
  if (!Array.isArray(queries) || queries.length > 10) {
    return res.status(400).json({ error: 'Invalid batch request' });
  }
  
  const results = await Promise.all(
    queries.map(query => tabulaLens.query(query))
  );
  
  res.json({ results });
});
```

### Response Optimization

#### 1. Use JSON Streaming for Large Datasets

```javascript
app.get('/api/tabula-lens/stream', async (req, res) => {
  res.setHeader('Content-Type', 'application/json');
  res.setHeader('Transfer-Encoding', 'chunked');
  
  res.write('{"data":[');
  
  const stream = await tabulaLens.queryStream(req.query);
  let first = true;
  
  for await (const row of stream) {
    if (!first) res.write(',');
    res.write(JSON.stringify(row));
    first = false;
  }
  
  res.write(']}');
  res.end();
});
```

#### 2. Implement Field Selection

```javascript
app.get('/api/tabula-lens', async (req, res) => {
  const { fields } = req.query;
  
  let query = req.query;
  if (fields) {
    query.columns = fields.split(',');
  }
  
  const result = await tabulaLens.query(query);
  res.json(result);
});
```

## Frontend Performance

### React.memo Optimization

The DatabaseViewer component and its sub-components use React.memo to prevent unnecessary re-renders.

```jsx
import { DatabaseViewer } from '@tabula-lens/react';

// Memoize parent component to prevent re-renders
const MyComponent = React.memo(({ data }) => {
  return (
    <DatabaseViewer
      endpoint="/api/tabula-lens"
      // Props that don't change frequently
    />
  );
});
```

### Data Fetching Optimization

#### 1. Use TanStack Query for Caching

```jsx
import { useQuery } from '@tanstack/react-query';

function useDatabaseData(query) {
  return useQuery({
    queryKey: ['database', query],
    queryFn: async () => {
      const response = await fetch('/api/tabula-lens?' + new URLSearchParams(query));
      return response.json();
    },
    staleTime: 5 * 60 * 1000,  // 5 minutes
    cacheTime: 10 * 60 * 1000,  // 10 minutes
  });
}
```

#### 2. Implement Debounced Filtering

```jsx
import { useDebouncedValue } from '@tabula-lens/react';

function DatabaseViewerWithDebounce() {
  const [filter, setFilter] = useState('');
  const debouncedFilter = useDebouncedValue(filter, 300);  // 300ms debounce
  
  return (
    <DatabaseViewer
      endpoint="/api/tabula-lens"
      filter={debouncedFilter}
      onFilterChange={setFilter}
    />
  );
}
```

#### 3. Prefetch Data

```jsx
import { useQueryClient } from '@tanstack/react-query';

function App() {
  const queryClient = useQueryClient();
  
  useEffect(() => {
    // Prefetch likely next page
    queryClient.prefetchQuery({
      queryKey: ['database', { table: 'users', page: 2 }],
      queryFn: async () => {
        const response = await fetch('/api/tabula-lens?table=users&page=2');
        return response.json();
      }
    });
  }, []);
  
  return <DatabaseViewer endpoint="/api/tabula-lens" />;
}
```

### Virtual Scrolling

For large datasets, implement virtual scrolling to render only visible rows.

```jsx
import { useVirtualizer } from '@tanstack/react-virtual';

function VirtualizedTable({ data }) {
  const parentRef = useRef();
  
  const virtualizer = useVirtualizer({
    count: data.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 50,  // Estimated row height
    overscan: 5  // Render 5 extra rows
  });
  
  return (
    <div ref={parentRef} style={{ height: '500px', overflow: 'auto' }}>
      <div style={{ height: `${virtualizer.getTotalSize()}px` }}>
        {virtualizer.getVirtualItems().map(virtualRow => (
          <div
            key={virtualRow.key}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              height: `${virtualRow.size}px`,
              transform: `translateY(${virtualRow.start}px)`
            }}
          >
            {data[virtualRow.index]}
          </div>
        ))}
      </div>
    </div>
  );
}
```

### Bundle Optimization

#### 1. Code Splitting

```jsx
import { lazy, Suspense } from 'react';

const DatabaseViewer = lazy(() => import('@tabula-lens/react'));

function App() {
  return (
    <Suspense fallback={<LoadingSpinner />}>
```

```javascript
// Import only what you need
import { DatabaseViewer } from '@tabula-lens/react';

// Instead of

```

### Database Query Profiling

```sql

ALTER DATABASE your_database SET log_min_duration_statement = 100;  -- Log queries > 100ms

-- Analyze slow queries
SELECT * FROM pg_stat_statements 
ORDER BY mean_exec_time DESC 
LIMIT 10;
```

### Backend Performance Monitoring

```javascript
const promClient = require('prom-client');

const httpRequestDuration = new promClient.Histogram({
  name: 'http_request_duration_seconds',
  help: 'Duration of HTTP requests in seconds',
  labelNames: ['method', 'route', 'status_code']
});

app.use((req, res, next) => {
  const start = Date.now();
  res.on('finish', () => {
    const duration = (Date.now() - start) / 1000;
    httpRequestDuration.observe(
      { method: req.method, route: req.path, status_code: res.statusCode },
      duration
    );
  });
  next();
});
```

## Performance Best Practices

### 1. Database

- ✅ Use appropriate indexes on frequently queried columns
- ✅ Implement connection pooling
- ✅ Use LIMIT to prevent large result sets
- ✅ Cache frequently accessed data
- ❌ Avoid SELECT * in production
- ❌ Don't create indexes on columns with low cardinality
- ❌ Avoid N+1 query patterns

### 2. Backend API

- ✅ Implement response compression
- ✅ Use caching for expensive operations
- ✅ Implement rate limiting
- ✅ Monitor and log performance metrics
- ❌ Avoid synchronous operations in request handlers
- ❌ Don't return more data than needed
- ❌ Avoid blocking the event loop

### 3. Frontend

- ✅ Use React.memo for expensive components
- ✅ Implement debouncing for user input
- ✅ Use virtual scrolling for large lists
- ✅ Implement code splitting
- ❌ Avoid unnecessary re-renders
- ❌ Don't fetch data on every keystroke
- ❌ Avoid rendering large DOM trees

## Troubleshooting

### Slow Database Queries

**Symptom:** API responses are slow

**Diagnosis:**
```sql
-- Check slow queries
SELECT query, mean_exec_time, calls 
FROM pg_stat_statements 
ORDER BY mean_exec_time DESC 
LIMIT 10;
```

**Solution:**
- Add appropriate indexes
- Optimize query structure
- Implement caching
- Consider database scaling

### High Memory Usage

**Symptom:** Application runs out of memory

**Diagnosis:**
```javascript
// Monitor memory usage
console.log('Memory usage:', process.memoryUsage());
```

**Solution:**
- Implement connection pooling
- Limit result set sizes
- Use streaming for large datasets
- Implement proper cache eviction

### Frontend Performance Issues

**Symptom:** UI is slow or unresponsive

**Diagnosis:**
```javascript
// Use React DevTools Profiler
// Check Chrome DevTools Performance tab
```

**Solution:**
- Implement React.memo
- Use virtual scrolling
- Optimize bundle size
- Implement code splitting
