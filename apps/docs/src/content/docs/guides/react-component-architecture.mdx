# React Component Architecture

The DatabaseViewer component is built with a modular, composable architecture that promotes reusability, maintainability, and performance optimization. This guide explains the component architecture, sub-component composition patterns, custom hook usage, and performance optimization strategies.

## Architecture Overview

The component follows a hierarchical architecture with clear separation of concerns:

```
DatabaseViewer (Main Orchestrator)
├── Custom Hooks
│   ├── useLogger
│   ├── useTableState
│   ├── useDatabaseData
│   └── buildQueryParams
├── Sub-components
│   ├── LoadingState
│   ├── ErrorState
│   ├── EmptyState
│   ├── TableSelector
│   ├── FilterInput
│   ├── Pagination
│   ├── DataTable
│   └── FilterColumnSelector
└── Utility Functions
    ├── fetchHelpers
    ├── validationHelpers
    ├── styleHelpers
    └── propValidation
```

## Main Component: DatabaseViewer

The DatabaseViewer component serves as the orchestrator that:

- Manages overall component state
- Coordinates data fetching through custom hooks
- Handles error states and retry logic
- Renders appropriate sub-components based on state
- Provides configuration through props
- Implements performance optimizations with React.memo

### Key Responsibilities

```tsx
// Simplified architecture overview
function DatabaseViewer(props: DatabaseViewerProps) {
  // Custom hooks for state management
  const logger = useLogger();
  const tableState = useTableState(props);
  const { data, loading, error, refetch } = useDatabaseData(props, tableState);
  
  // Render appropriate sub-components based on state
  if (loading) return <LoadingState />;
  if (error) return <ErrorState error={error} retry={refetch} />;
  if (!data || data.rows.length === 0) return <EmptyState />;
  
  return (
    <>
      <TableSelector />
      <FilterInput />
      <DataTable />
      <Pagination />
    </>
  );
}
```

## Custom Hooks

### useLogger Hook

Provides logging functionality with configurable levels.

**Features:**
- Configurable log levels (debug, info, warn, error, silent)
- Browser-compatible logging
- Request ID tracking
- Performance monitoring

**Usage:**

```tsx
import { useLogger } from '@tabula-lens/react';

function MyComponent() {
  const logger = useLogger();
  
  useEffect(() => {
    logger.info('Component mounted');
    logger.debug('Debug information');
    logger.warn('Warning message');
    logger.error('Error occurred');
  }, []);
  
  return <div>My Component</div>;
}
```

### useTableState Hook

Manages table state (current table, sorting, filtering, pagination).

**Features:**
- Centralized state management
- Memoized state updates
- Type-safe state operations
- Performance optimized

**Usage:**

```tsx
import { useTableState } from '@tabula-lens/react';

function MyComponent() {
  const tableState = useTableState({
    initialTable: 'users',
    defaultSort: { column: 'id', direction: 'asc' },
    pageSize: 20
  });
  
  const { currentTable, sort, filter, pagination } = tableState;
  
  return <div>Current table: {currentTable}</div>;
}
```

### useDatabaseData Hook

Handles data fetching with caching and error handling.

**Features:**
- Automatic data fetching
- Error handling and retry logic
- Loading state management
- Refetch capabilities
- Cache management

**Usage:**

```tsx
import { useDatabaseData } from '@tabula-lens/react';

function MyComponent() {
  const { data, loading, error, refetch } = useDatabaseData({
    path: '/api/tabula-lens',
    table: 'users',
    sort: { column: 'id', direction: 'asc' },
    pagination: { page: 1, pageSize: 20 }
  });
  
  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;
  
  return <div>Data: {JSON.stringify(data)}</div>;
}
```

### buildQueryParams Function

Builds query parameters from table state.

**Features:**
- Type-safe parameter building
- Handles complex state objects
- URL encoding
- Filter serialization

**Usage:**

```tsx
import { buildQueryParams } from '@tabula-lens/react';

function MyComponent() {
  const params = buildQueryParams({
    table: 'users',
    sort: { column: 'id', direction: 'asc' },

```

## Performance Optimization

### React.memo Usage

All sub-components use React.memo to prevent unnecessary re-renders:

```tsx
export const LoadingState = memo(function LoadingState({ message }: LoadingStateProps) {
  return <div className="loading">{message}</div>;
});
```

### Memoized Render Functions

The main component uses memoized render functions for expensive operations:

```tsx
const TableSelectorRenderer = useMemo(() => (
  <TableSelector 
    tables={tables}
    currentTable={currentTable}
    onTableChange={handleTableChange}
  />
), [tables, currentTable, handleTableChange]);
```

### useMemo for Style Calculations

Style calculations are memoized to avoid recomputation:

```tsx
const containerStyle = useMemo(() => ({
  ...defaultStyles.container,
  ...customStyles.container
}), [customStyles.container]);
```

### Pagination Memoization

Pagination calculations are memoized for performance:

```tsx
const paginationInfo = useMemo(() => ({
  totalPages: Math.ceil(totalRows / pageSize),
  hasNextPage: currentPage < totalPages,
  hasPreviousPage: currentPage > 1
}), [totalRows, pageSize, currentPage]);
```

## State Management Patterns

### Centralized State

The component uses centralized state management through custom hooks:

```tsx
function DatabaseViewer(props) {
  const tableState = useTableState(props);
  const { data, loading, error } = useDatabaseData(props, tableState);
  
  // All state is managed through hooks
  // Sub-components receive only the data they need
}
```

### Prop Drilling Minimization

The component minimizes prop drilling by:

- Using custom hooks for state management
- Composing sub-components with minimal props
- Leveraging React context when appropriate
- Using render props for complex customization

### Event Handling

Event handlers are memoized to prevent unnecessary re-renders:

```tsx
const handleTableChange = useCallback((table: string) => {
  setCurrentTable(table);
}, []);
```

## Best Practices

1. **Use Custom Components for Complex UI**: Replace default sub-components when you need complex custom UI
2. **Leverage Custom Hooks**: Use exported custom hooks for advanced state management
3. **Optimize Performance**: Use React.memo and useMemo for expensive operations
4. **Type Safety**: Leverage TypeScript definitions for type-safe component composition
5. **Design System Integration**: Use CSS custom properties for consistent theming
6. **Error Handling**: Implement proper error boundaries and error callbacks
7. **Testing**: Test custom components and hooks independently

## Migration Guide

If you're migrating from an older version:

1. **Component Props**: All props remain backward compatible
2. **Custom Components**: Custom component patterns are enhanced, not changed
3. **Performance**: React.memo is automatically applied, no changes needed
4. **Styling**: CSS custom properties are now available for theming
5. **Hooks**: Custom hooks are now exported for advanced use cases

The modular architecture ensures that you can adopt new features incrementally without breaking existing functionality.
