# Database Support

Tabula Lens supports four major SQL database engines: PostgreSQL, MySQL, SQLite, and SQL Server. This guide covers connection patterns, configuration, and best practices that apply across all supported databases, with database-specific details for each engine.

For detailed setup instructions for a specific database, see the dedicated guides:

- [MySQL Support](./mysql-support)
- [SQLite Support](./sqlite-support)
- [SQL Server Support](./mssql-support)

PostgreSQL-specific details are included in the sections below and in the integration guides for managed PostgreSQL providers.

## Supported Databases

| Database | Knex client | Driver package | Best for |
|----------|-------------|----------------|----------|
| PostgreSQL / CockroachDB | `pg` | `pg` | Production applications, complex queries, JSON support |
| MySQL / MariaDB | `mysql2` | `mysql2` | Web applications, CMS, LAMP stacks |
| SQLite | `better-sqlite3` | `better-sqlite3` | Embedded databases, development, testing, single-file storage |
| SQL Server (MSSQL) | `mssql` | `tedious` | Enterprise applications, Microsoft ecosystems |

## Installation

Install `@tabula-lens/node` and the database driver for your chosen engine:

<PackageManagers pkg="@tabula-lens/node pg" comment="# PostgreSQL / CockroachDB" />
<PackageManagers pkg="@tabula-lens/node mysql2" comment="# MySQL / MariaDB" />
<PackageManagers pkg="@tabula-lens/node better-sqlite3" comment="# SQLite" />

Database drivers are peer dependencies. Only the driver for your chosen database needs to be installed.

## Connection Patterns

Tabula Lens accepts either a database URL string or a config object. The database type is automatically detected from the URL, or you can specify it explicitly with the `type` field.

### String Form (Existing Behavior)

```typescript
import { TabulaLens } from '@tabula-lens/node';

const tabulaLens = new TabulaLens(process.env.DATABASE_URL);
```

### Config Object Form

```typescript
import { TabulaLens } from '@tabula-lens/node';

const tabulaLens = new TabulaLens({
  url: process.env.DATABASE_URL,
  type: 'mysql', // optional — auto-detected from URL
  logLevel: 'info',
  sensitiveDataMasking: true,
});
```

### Auto-Detection Reference

| URL prefix / pattern | Detected type |
|----------------------|---------------|
| `postgresql://`, `postgres://`, `pgsql://` | `pg` |
| `mysql://`, `mysql2://`, `mysqlx://`, `mariadb://` | `mysql` |
| `sqlite://`, `sqlite:`, `file:`, `:memory:`, paths ending in `.db`/`.sqlite`/`.sqlite3`/`.db3` | `sqlite` |
| `mssql://`, `sqlserver://`, `mssql+tcp://`, `mssql+udp://` | `mssql` |

If the type cannot be detected, Tabula Lens throws a `TabulaLensError` with the message:

```
Unable to detect database type from URL. Please specify the 'type' field in the config. Valid types: pg, mysql, sqlite, mssql
```

### Production Configuration

```typescript
import { TabulaLens } from '@tabula-lens/node';

const tabulaLens = new TabulaLens(process.env.DATABASE_URL, {
  logLevel: 'error',
  logFormat: 'json',
  sensitiveDataMasking: true,
  enableRequestLogging: true,
});
```

### With Custom Logger

```typescript
import { TabulaLens } from '@tabula-lens/node';
import winston from 'winston';

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.Console(),
    new winston.transports.File({ filename: 'database.log' }),
  ],
});

const tabulaLens = new TabulaLens(process.env.DATABASE_URL, {
  logger: {
    debug: (msg) => logger.debug(msg),
    info: (msg) => logger.info(msg),
    warn: (msg) => logger.warn(msg),
    error: (msg) => logger.error(msg),
  },
});
```

## PostgreSQL Subsection

### Connection String Format

```bash
DATABASE_URL="postgresql://user:password@localhost:5432/mydb"
```

```
postgresql://[user[:password]@][host][:port][/database][?parameters]
```

### Database Setup

```sql
-- Create database
CREATE DATABASE tabula_lens_db;

-- Create schema
CREATE SCHEMA IF NOT EXISTS public;

-- Create users table
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### User Permissions

```sql
-- Create read-only user
CREATE USER tabula_lens_readonly WITH PASSWORD 'secure_password';
GRANT CONNECT ON DATABASE tabula_lens_db TO tabula_lens_readonly;
GRANT USAGE ON SCHEMA public TO tabula_lens_readonly;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO tabula_lens_readonly;
ALTER DEFAULT PRIVILEGES IN SCHEMA public
  GRANT SELECT ON TABLES TO tabula_lens_readonly;
```

### Connection Parameters

```bash
# SSL mode
DATABASE_URL="postgresql://user:password@localhost:5432/mydb?sslmode=require"

# Connection pool settings
DATABASE_URL="postgresql://user:password@localhost:5432/mydb?pool_min=2&pool_max=10"

# Statement timeout
DATABASE_URL="postgresql://user:password@localhost:5432/mydb?statement_timeout=30000"
```

## Connection Pooling

Tabula Lens relies on Knex.js for connection pooling. Pool settings are typically controlled through connection string parameters or database-specific configuration, not through the TabulaLens constructor.

### Example Pool Configuration

```bash
# PostgreSQL / MySQL / SQL Server
DATABASE_URL="postgresql://user:password@localhost:5432/mydb?pool_min=2&pool_max=10"
```

SQLite uses `better-sqlite3`, which does not use a traditional connection pool because it operates on a single database file.

## Security Best Practices

### Environment Variables

Always use environment variables for connection strings:

```bash
# .env file
DATABASE_URL=postgresql://user:password@localhost:5432/mydb
```

```typescript
import dotenv from 'dotenv';
dotenv.config();

const tabulaLens = new TabulaLens(process.env.DATABASE_URL);
```

### SSL Connections

Use SSL in production when supported by your database:

```bash
# PostgreSQL
DATABASE_URL="postgresql://user:password@host:5432/mydb?sslmode=require"

# MySQL
DATABASE_URL="mysql://user:password@host:3306/mydb?ssl=true"

# SQL Server
DATABASE_URL="mssql://user:password@host:1433/mydb?encrypt=true"
```

### Least Privilege

Create a dedicated database user with minimal permissions for Tabula Lens. See the database-specific guides for example SQL.

## Query Optimization

### Select Only Needed Columns

```typescript
const result = await tabulaLens.query({
  table: 'users',
  columns: ['id', 'name', 'email'],
  page: 1,
  limit: 20,
});
```

### Use Pagination

```typescript
const result = await tabulaLens.query({
  table: 'users',
  page: 1,
  limit: 50,
});
```

### Add Indexes

Create indexes on columns used for filtering, sorting, and joining:

```sql
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_created_at ON users(created_at);
```

## Choosing a Database

| Use Case | Recommended Database |
|----------|---------------------|
| Production web application with complex queries | PostgreSQL |
| Existing MySQL/MariaDB infrastructure | MySQL |
| Local development, testing, or single-file storage | SQLite |
| Enterprise environment using Microsoft stack | SQL Server |
| JSON/JSONB-heavy workloads | PostgreSQL |
| Serverless or edge deployments requiring TCP connection | PostgreSQL, MySQL, or SQL Server |

## Troubleshooting

### Auto-Detection Failures

**Problem:** Tabula Lens cannot detect the database type from the URL.

**Solution:** Specify the `type` field explicitly:

```typescript
const tabulaLens = new TabulaLens({
  url: process.env.DATABASE_URL,
  type: 'mysql',
});
```

### Driver Installation Issues

**Problem:** Error message mentions missing driver module.

**Solution:** Install the correct peer dependency:

<PackageManagers pkg="pg" comment="# For PostgreSQL" />
<PackageManagers pkg="mysql2" comment="# For MySQL" />
<PackageManagers pkg="better-sqlite3" comment="# For SQLite" />
<PackageManagers pkg="tedious" comment="# For SQL Server" />

### Connection Issues

1. Verify the connection string format for your database engine.
2. Check that the database server is running and reachable.
3. Verify firewall rules allow connections on the correct port.
4. Confirm the database user has the necessary permissions.

For database-specific troubleshooting, see the dedicated support guides.

## Best Practices

1. **Use environment variables** for connection strings.
2. **Install only the driver you need** to keep your dependency tree small.
3. **Use read-only users** when the application only needs to query data.
4. **Implement proper indexing** for frequently queried columns.
5. **Use SSL connections** in production.
6. **Enable sensitive data masking** in production logs.
7. **Test auto-detection** if you rely on it, or specify `type` explicitly for clarity.
