# SQL Server Support

This guide covers how to connect Tabula Lens to your Microsoft SQL Server database with comprehensive configuration, optimization, and troubleshooting information.

## Supported Databases

Tabula Lens supports Microsoft SQL Server (MSSQL) databases through Knex.js using the `tedious` driver. Azure SQL Database and Amazon RDS for SQL Server are also compatible.

## Installation

Install the Tabula Lens Node package and the SQL Server driver:

```bash
npm install @tabula-lens/node tedious
```

## Database Connection

### Connection String Format

Set your database connection string as an environment variable:

```bash
DATABASE_URL="mssql://user:password@localhost:1433/mydb"
```

### Connection String Components

A SQL Server connection string has the following format:

```
mssql://[user[:password]@][host][:port][/database][?parameters]
```

**Components:**

- `user` - Database username
- `password` - Database password (optional)
- `host` - Database host (default: localhost)
- `port` - Database port (default: 1433)
- `database` - Database name
- `parameters` - Additional connection parameters

### Alternative Connection String Formats

SQL Server supports several connection string variations:

```bash
# Standard URL format
DATABASE_URL="mssql://user:password@localhost:1433/mydb"

# SQL Server-specific scheme
DATABASE_URL="sqlserver://user:password@localhost:1433/mydb"

# With TCP variant
DATABASE_URL="mssql+tcp://user:password@localhost:1433/mydb"
```

### Connection Parameters

Common connection parameters:

```bash
# Encrypt connection (recommended for production)
DATABASE_URL="mssql://user:password@localhost:1433/mydb?encrypt=true"

# Trust server certificate (development only)
DATABASE_URL="mssql://user:password@localhost:1433/mydb?trustServerCertificate=true"

# Connection timeout (milliseconds)
DATABASE_URL="mssql://user:password@localhost:1433/mydb?connectTimeout=15000"

# Request timeout (milliseconds)
DATABASE_URL="mssql://user:password@localhost:1433/mydb?requestTimeout=30000"

# Connection pool settings
DATABASE_URL="mssql://user:password@localhost:1433/mydb?pool_min=2&pool_max=10"
```

## Connection Patterns

### Local Development

```typescript
import { TabulaLens } from '@tabula-lens/node';

const tabulaLens = new TabulaLens(process.env.DATABASE_URL);

// Test connection
const tables = await tabulaLens.getTables();
console.log('Available tables:', tables);
```

### Production

```typescript
import { TabulaLens } from '@tabula-lens/node';

const tabulaLens = new TabulaLens(process.env.DATABASE_URL, {
  logLevel: 'error',
  logFormat: 'json',
  sensitiveDataMasking: true,
  enableRequestLogging: true,
});
```

### With Config Object

```typescript
import { TabulaLens } from '@tabula-lens/node';

const tabulaLens = new TabulaLens({
  url: process.env.DATABASE_URL,
  type: 'mssql',
  logLevel: 'info',
  logFormat: 'json',
  sensitiveDataMasking: true,
  enableRequestLogging: true,
});
```

### With Custom Logger

```typescript
import { TabulaLens } from '@tabula-lens/node';
import winston from 'winston';

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.Console(),
    new winston.transports.File({ filename: 'database.log' }),
  ],
});

const tabulaLens = new TabulaLens(process.env.DATABASE_URL, {
  logger: {
    debug: (msg) => logger.debug(msg),
    info: (msg) => logger.info(msg),
    warn: (msg) => logger.warn(msg),
    error: (msg) => logger.error(msg),
  },
});
```

## Connection Pooling

Tabula Lens automatically manages connection pooling through Knex.js for optimal performance.

### Default Pool Configuration

```typescript
// Default pool settings (handled by Knex.js)
{
  min: 2,
  max: 10,
}
```

### Custom Pool Configuration

```typescript
const tabulaLens = new TabulaLens(process.env.DATABASE_URL, {
  // Connection pool settings in connection string
  // DATABASE_URL="mssql://user:password@localhost:1433/mydb?pool_min=5&pool_max=20"
});
```

### Pool Configuration Options

```bash
# Minimum connections
DATABASE_URL="mssql://user:password@localhost:1433/mydb?pool_min=5"

# Maximum connections
DATABASE_URL="mssql://user:password@localhost:1433/mydb?pool_max=20"

# Idle timeout (milliseconds)
DATABASE_URL="mssql://user:password@localhost:1433/mydb?pool_idle_timeout=30000"
```

## Database Setup

### Creating a Database

```sql
-- Create database
CREATE DATABASE tabula_lens_db;

-- Use the database
USE tabula_lens_db;

-- Set recovery model (recommended for production)
ALTER DATABASE tabula_lens_db SET RECOVERY FULL;
```

### Creating Tables

```sql
-- Create users table
CREATE TABLE users (
  id INT IDENTITY(1,1) PRIMARY KEY,
  name NVARCHAR(255) NOT NULL,
  email NVARCHAR(255) UNIQUE NOT NULL,
  created_at DATETIME2 DEFAULT GETDATE(),
  updated_at DATETIME2 DEFAULT GETDATE()
);

-- Create products table
CREATE TABLE products (
  id INT IDENTITY(1,1) PRIMARY KEY,
  name NVARCHAR(255) NOT NULL,
  price DECIMAL(10, 2) NOT NULL,
  description NVARCHAR(MAX),
  created_at DATETIME2 DEFAULT GETDATE()
);

-- Create orders table
CREATE TABLE orders (
  id INT IDENTITY(1,1) PRIMARY KEY,
  user_id INT,
  total DECIMAL(10, 2) NOT NULL,
  status NVARCHAR(50) DEFAULT 'pending',
  created_at DATETIME2 DEFAULT GETDATE(),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### Adding Sample Data

```sql
-- Insert sample users
INSERT INTO users (name, email) VALUES
  ('John Doe', 'john@example.com'),
  ('Jane Smith', 'jane@example.com'),
  ('Bob Johnson', 'bob@example.com');

-- Insert sample products
INSERT INTO products (name, price, description) VALUES
  ('Product A', 29.99, 'Description for Product A'),
  ('Product B', 49.99, 'Description for Product B'),
  ('Product C', 19.99, 'Description for Product C');

-- Insert sample orders
INSERT INTO orders (user_id, total, status) VALUES
  (1, 29.99, 'completed'),
  (2, 49.99, 'pending'),
  (1, 19.99, 'shipped');
```

## User Permissions

### Creating Read-Only User

```sql
-- Create login at server level
CREATE LOGIN tabula_lens_readonly WITH PASSWORD = 'secure_password';

-- Create user in the database
USE tabula_lens_db;
CREATE USER tabula_lens_readonly FOR LOGIN tabula_lens_readonly;

-- Grant select permission
ALTER ROLE db_datareader ADD MEMBER tabula_lens_readonly;
```

### Creating Read-Write User

```sql
-- Create login at server level
CREATE LOGIN tabula_lens_user WITH PASSWORD = 'secure_password';

-- Create user in the database
USE tabula_lens_db;
CREATE USER tabula_lens_user FOR LOGIN tabula_lens_user;

-- Grant read and write permissions
ALTER ROLE db_datareader ADD MEMBER tabula_lens_user;
ALTER ROLE db_datawriter ADD MEMBER tabula_lens_user;
```

### Schema-Scoped Permissions

For finer-grained control, grant permissions on specific schemas:

```sql
USE tabula_lens_db;

-- Create custom role
CREATE ROLE tabula_lens_reader;

-- Grant select on schema
GRANT SELECT ON SCHEMA::dbo TO tabula_lens_reader;

-- Add user to role
ALTER ROLE tabula_lens_reader ADD MEMBER tabula_lens_readonly;
```

## Security Best Practices

### Environment Variables

Use environment variables for connection strings:

```bash
# .env file
DATABASE_URL=mssql://user:password@localhost:1433/mydb
```

```typescript
// Load environment variables

```

### SSL/TLS Connections

Use encrypted connections in production:

```bash
# Require encryption
DATABASE_URL="mssql://user:password@localhost:1433/mydb?encrypt=true"

# Use strict encryption (Azure SQL recommended)
DATABASE_URL="mssql://user:password@localhost:1433/mydb?encrypt=true&trustServerCertificate=false"
```

### Connection Security

```typescript
// Production configuration with security
const tabulaLens = new TabulaLens(process.env.DATABASE_URL, {
  logLevel: 'error',
  logFormat: 'json',
  sensitiveDataMasking: true,
  enableRequestLogging: true,

```

### Network Security

- Use firewall rules to restrict database access
- Use VPNs or private endpoints for remote database access
- Implement IP whitelisting for database connections
- Use strong passwords for database users
- Rotate database credentials regularly
- Disable the `sa` account or use a strong, rotated password
- Enable SQL Server audit logging

## Query Optimization

### Indexing

Create indexes for frequently queried columns:

```sql
-- Create index on email column
CREATE INDEX idx_users_email ON users(email);

-- Create index on created_at column
CREATE INDEX idx_users_created_at ON users(created_at);

-- Create composite index
CREATE INDEX idx_orders_user_status ON orders(user_id, status);

-- Create full-text index for text search
CREATE FULLTEXT CATALOG ftCatalog AS DEFAULT;
CREATE FULLTEXT INDEX ON products(description)
  KEY INDEX PK__products__3213E83F00000000; -- replace with actual primary key index name
```

### Query Performance

Optimize queries for better performance:

```typescript
// Select only needed columns
const result = await tabulaLens.query({
  table: 'users',
  columns: ['id', 'name', 'email'], // Only select needed columns
  page: 1,
  limit: 20,
});

// Use appropriate page sizes
const result = await tabulaLens.query({
  table: 'users',
  page: 1,
  limit: 50, // Balance between data transfer and number of requests
});

// Use filtering to reduce data transfer
const result = await tabulaLens.query({
  table: 'users',
  filter: 'john', // Filter results using LIKE
  page: 1,
  limit: 20,
});
```

### Database Configuration

Optimize SQL Server configuration:

```sql
-- Increase max server memory (MB)
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;
EXEC sp_configure 'max server memory (MB)', 4096;
RECONFIGURE;

-- Enable query store for workload analysis
ALTER DATABASE tabula_lens_db SET QUERY_STORE = ON;

-- Set compatibility level (use latest supported)
ALTER DATABASE tabula_lens_db SET COMPATIBILITY_LEVEL = 160; -- SQL Server 2022
```

### Filter Behavior

SQL Server filtering in Tabula Lens uses the `LIKE` operator. SQL Server `LIKE` is case-insensitive when the database collation is case-insensitive, which is the default for most installations. If your collation is case-sensitive (`_CS_`), filtering will also be case-sensitive.

## Database-Specific Features and Limitations

### Supported Text Types

Tabula Lens recognizes the following SQL Server text types for filtering:

- `varchar`
- `nvarchar`
- `text`
- `ntext`
- `char`
- `nchar`

Non-text types such as `INT`, `DECIMAL`, `DATETIME2`, `UNIQUEIDENTIFIER`, `VARBINARY`, and `XML` are not included in global filter searches unless explicitly selected with `filterColumns`.

### Unicode Support

Use `NVARCHAR`, `NCHAR`, and `NTEXT` for full Unicode support. When inserting Unicode literals, prefix strings with `N`:

```sql
INSERT INTO users (name, email) VALUES (N'José García', N'jose@example.com');
```

Tabula Lens filters using `LIKE` work correctly with Unicode data in `NVARCHAR` columns.

### JSON Data Types

SQL Server 2016+ supports JSON functions. You can store JSON in `NVARCHAR` columns:

```sql
-- Create table with JSON stored as NVARCHAR
CREATE TABLE user_profiles (
  id INT IDENTITY(1,1) PRIMARY KEY,
  user_id INT,
  profile NVARCHAR(MAX),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Insert JSON data
INSERT INTO user_profiles (user_id, profile) VALUES
  (1, '{"age": 30, "city": "New York", "interests": ["coding", "music"]}');
```

Tabula Lens reads JSON values as text but does not automatically search inside JSON documents with the global filter.

### Metadata Queries

SQL Server metadata is read from `information_schema.tables` and `information_schema.columns`, filtered to base tables. Views are excluded from the table list returned by `getTables()`.

### Known Limitations

- Tabula Lens does not generate or alter SQL Server views through the query API.
- Stored procedures and functions are not exposed through the standard query interface.
- Full-text search requires explicit indexes and is not used by the default global filter, which uses `LIKE`.
- The `tedious` driver may require additional options for complex authentication modes such as Azure AD.

## Monitoring and Observability

### Query Logging

Enable query logging for monitoring:

```typescript
const tabulaLens = new TabulaLens(process.env.DATABASE_URL, {
  logLevel: 'debug',
  enableQueryLogging: true,
  logFormat: 'json',
});
```

### Performance Monitoring

Monitor database performance:

```sql
-- Enable Query Store
ALTER DATABASE tabula_lens_db SET QUERY_STORE = ON;

-- View top resource-consuming queries
SELECT TOP 10
  qst.query_sql_text,
  qsrs.avg_duration / 1000 AS avg_duration_ms
FROM sys.query_store_query_text qst
JOIN sys.query_store_query qs ON qst.query_text_id = qs.query_text_id
JOIN sys.query_store_plan qp ON qs.query_id = qp.query_id
JOIN sys.query_store_runtime_stats qsrs ON qp.plan_id = qsrs.plan_id
ORDER BY qsrs.avg_duration DESC;
```

### Dynamic Management Views

```sql
-- View active connections
SELECT
  session_id,
  login_name,
  status,
  host_name,
  program_name
FROM sys.dm_exec_sessions
WHERE is_user_process = 1;

-- View currently running requests
SELECT
  session_id,
  start_time,
  status,
  command,
  total_elapsed_time
FROM sys.dm_exec_requests
WHERE session_id > 50;
```

### Connection Monitoring

```sql
-- View maximum connections
SELECT @@MAX_CONNECTIONS AS max_connections;

-- View current user connections
SELECT COUNT(*) AS user_connections
FROM sys.dm_exec_sessions
WHERE is_user_process = 1;
```

## Backup and Recovery

### Database Backup

```sql
-- Full database backup
BACKUP DATABASE tabula_lens_db
  TO DISK = '/var/backups/tabula_lens_db_full.bak';

-- Differential backup
BACKUP DATABASE tabula_lens_db
  TO DISK = '/var/backups/tabula_lens_db_diff.bak'
  WITH DIFFERENTIAL;

-- Transaction log backup
BACKUP LOG tabula_lens_db
  TO DISK = '/var/backups/tabula_lens_db_log.trn';
```

### Database Restore

```sql
-- Restore database from full backup
RESTORE DATABASE tabula_lens_db
  FROM DISK = '/var/backups/tabula_lens_db_full.bak'
  WITH REPLACE;

-- Restore with differential and log backups
RESTORE DATABASE tabula_lens_db
  FROM DISK = '/var/backups/tabula_lens_db_full.bak'
  WITH NORECOVERY;

RESTORE DATABASE tabula_lens_db
  FROM DISK = '/var/backups/tabula_lens_db_diff.bak'
  WITH NORECOVERY;

RESTORE LOG tabula_lens_db
  FROM DISK = '/var/backups/tabula_lens_db_log.trn'
  WITH RECOVERY;
```

### Recovery Model

Use the `FULL` recovery model for production to enable point-in-time recovery:

```sql
ALTER DATABASE tabula_lens_db SET RECOVERY FULL;
```

## Troubleshooting

### Connection Issues

If you encounter connection issues:

```typescript
// Verify connection string
console.log('DATABASE_URL:', process.env.DATABASE_URL);

// Test connection
try {
  const tables = await tabulaLens.getTables();
  console.log('Connected successfully. Tables:', tables);
} catch (error) {
  console.error('Connection failed:', error);
}
```

**Common connection issues:**

1. **Incorrect connection string**
   ```bash
   # Verify format
   mssql://user:password@host:port/database
   ```

2. **Database server not running**
   ```powershell
   # Check SQL Server status on Windows
   Get-Service MSSQLSERVER
   ```

3. **Network connectivity issues**
   ```bash
   # Test network connectivity
   ping localhost
   telnet localhost 1433
   ```

4. **Authentication issues**
   ```sql
   -- Verify user permissions
   SELECT name, type_desc, is_disabled FROM sys.sql_logins;
   SELECT dp.name, dp.type_desc
   FROM sys.database_principals dp
   JOIN sys.database_role_members drm ON dp.principal_id = drm.member_principal_id
   JOIN sys.database_principals rp ON drm.role_principal_id = rp.principal_id
   WHERE rp.name IN ('db_datareader', 'db_datawriter');
   ```

### Performance Issues

For performance optimization:

1. **Check query performance**
   ```sql
   SET STATISTICS TIME ON;
   SET STATISTICS IO ON;
   SELECT * FROM users WHERE email = 'john@example.com';
   ```

2. **Check index usage**
   ```sql
   SELECT
     OBJECT_NAME(object_id) AS table_name,
     name AS index_name,
     user_seeks,
     user_scans,
     user_lookups,
     user_updates
   FROM sys.dm_db_index_usage_stats
   WHERE database_id = DB_ID('tabula_lens_db');
   ```

3. **Check table statistics**
   ```sql
   UPDATE STATISTICS users;
   DBCC SHOW_STATISTICS('users', 'idx_users_email');
   ```

4. **Check database size**
   ```sql
   SELECT
     DB_NAME(database_id) AS database_name,
     name AS logical_name,
     type_desc,
     size / 128.0 AS size_mb
   FROM sys.database_files
   WHERE database_id = DB_ID('tabula_lens_db');
   ```

### Memory Issues

If you encounter memory issues:

1. **Increase max server memory**
   ```sql
   EXEC sp_configure 'show advanced options', 1;
   RECONFIGURE;
   EXEC sp_configure 'max server memory (MB)', 8192;
   RECONFIGURE;
   ```

2. **Clear procedure cache (use with caution)**
   ```sql
   DBCC FREEPROCCACHE;
   ```

3. **Check buffer pool usage**
   ```sql
   SELECT
     (COUNT(*) * 8) / 1024.0 AS buffer_pool_mb
   FROM sys.dm_os_buffer_descriptors
   WHERE database_id = DB_ID('tabula_lens_db');
   ```

### Lock Issues

If you encounter lock issues:

```sql
-- View active locks
SELECT
  request_session_id,
  resource_type,
  resource_description,
  request_mode,
  request_status
FROM sys.dm_tran_locks;

-- View blocked sessions
SELECT
  blocked.session_id AS blocked_session,
  blocking.session_id AS blocking_session,
  blocked.text AS blocked_query,
  blocking.text AS blocking_query
FROM (SELECT session_id, text FROM sys.dm_exec_requests CROSS APPLY sys.dm_exec_sql_text(sql_handle)) blocked
JOIN (SELECT session_id, text FROM sys.dm_exec_requests CROSS APPLY sys.dm_exec_sql_text(sql_handle)) blocking
  ON blocked.blocking_session_id = blocking.session_id;

-- Kill specific session
KILL <session_id>;
```
