# ORM Coexistence (Prisma & Drizzle)

Many applications use an ORM like Prisma or Drizzle for application data access and want a database viewer for debugging and admin use. Tabula Lens is designed to coexist with these tools without any conflicts.

## How It Works

Tabula Lens, Prisma, and Drizzle each maintain their own independent connection pool to the database. They do not share state or interfere with each other at the connection level. All three can use the same `DATABASE_URL`.

## With Prisma

Install Prisma and initialize your schema if you haven't already:

```bash
npm install @prisma/client prisma
npx prisma init
```

Initialize both clients in a shared module:

```typescript
// lib/db.ts
import { TabulaLens } from '@tabula-lens/node';
import { PrismaClient } from '@prisma/client';

// Tabula Lens — admin database viewer
export const tabulaLens = new TabulaLens(process.env.DATABASE_URL!);

// Prisma — application business logic
export const prisma = new PrismaClient();
```

Use each client for its intended purpose in your route handlers:

```typescript
// app/api/tabula-lens/route.ts — database viewer (admin only)
import { tabulaLens } from '@/lib/db';
import { createNextRouteHandler } from '@tabula-lens/node';

const handler = createNextRouteHandler(tabulaLens);
export const GET = handler;
```

```typescript
// app/api/users/route.ts — application logic
import { prisma } from '@/lib/db';

export async function GET() {
  const users = await prisma.user.findMany();
  return Response.json(users);
}
```

## With Drizzle

Install Drizzle and its PostgreSQL driver:

```bash
npm install drizzle-orm postgres
```

Initialize both clients in a shared module:

```typescript
// lib/db.ts
import { TabulaLens } from '@tabula-lens/node';
import { drizzle } from 'drizzle-orm/postgres-js';
import postgres from 'postgres';

// Tabula Lens — admin database viewer
export const tabulaLens = new TabulaLens(process.env.DATABASE_URL!);

// Drizzle — application business logic
const queryClient = postgres(process.env.DATABASE_URL!);
export const db = drizzle(queryClient);
```

The Tabula Lens route setup is identical to the Prisma example above — import `tabulaLens` from `lib/db.ts` and pass it to your framework's adapter.

## Connection Pool Considerations

When running both an ORM and Tabula Lens, multiple connection pools are open to the same database simultaneously. The total connection count is the sum of all pool sizes. For most applications this is not a concern, but if you are on a plan with a low connection limit (Neon free tier, Supabase free tier):

- Use a pooled connection string as the `DATABASE_URL` for all clients. Neon's built-in pooler, Supabase Supavisor, and self-hosted PgBouncer all work.
- Reduce the pool `min` and `max` values in each client's configuration to stay within your limit.

## Recommended Use Case Split

| Tool | Use for |
|---|---|
| Tabula Lens | Admin database viewer, debugging, data inspection |
| Prisma / Drizzle | Application business logic, mutations, typed queries |

Each tool is optimized for its role. Tabula Lens provides a read-oriented admin interface; your ORM handles the application's typed data access patterns.
