# Production Guides

These guides cover deploying, optimizing, and maintaining Tabula Lens in production environments.

## Guides

- **[Deployment](./deployment/)** - Deployment strategies for database, backend, and frontend
- **[Performance Optimization](./performance-optimization/)** - Optimizing performance across the stack
- **[Testing](./testing/)** - Testing strategies and best practices

## Deployment Options

Tabula Lens can be deployed to various platforms:

- **Database** - Managed PostgreSQL (Supabase, Neon, AWS RDS) or self-hosted
- **Backend** - Vercel, Railway, AWS EC2, Docker, Kubernetes
- **Frontend** - Vercel, Netlify, AWS S3, Docker

See the [Deployment](./deployment/) guide for detailed instructions for each platform.

## Performance Optimization

Performance optimization covers:

- Database indexing and query optimization
- API response optimization
- Frontend rendering performance
- Bundle optimization
- Caching strategies

## Testing

The testing guide covers:

- Integration testing
- End-to-end testing patterns
- CI/CD testing configuration
